# ShareStamps 제품 비전 (VISION)

> 이 문서는 ShareStamps가 **왜** 이렇게 설계되었는지를 담는다. 여러 AI 도구
> (Antigravity → Codex → Claude Code)가 순차로 작업하므로, 각 기능의 *의도*를
> 잃지 않도록 비즈니스 로직 전체를 한 곳에 기록한다. 구현 방식은 코드와
> [CLAUDE.md](../CLAUDE.md)를, **목적**은 이 문서를 본다.

## 한 문장 요약

소규모 매장의 단골 손님을 **AI 시대에 맞는 전방위 콘텐츠(AEO/GEO) 생산자**로
전환하고, **스탬프 보상**을 사회적 선행(기부)·관계(선물)와 묶어, **직원
인센티브**로 가동시키는 **선순환 마케팅 엔진**.

---

## A. 출발점 — 문제 정의 (왜 존재하는가)

- **A1.** 소규모 매장의 생존은 결국 **홍보·마케팅**에 달려 있다.
- **A2.** 지금은 **AI 시대** — 검색 결과를 클릭하지 않는 **노클릭 서치(no-click search)** 가 늘었다.
- **A3.** 그래서 **SEO는 기본**, 이제는 **AEO(답변엔진 최적화) + GEO(생성엔진 최적화)** 를 해야 한다.
- **A4.** 한 곳에만 콘텐츠를 올려선 안 된다 → **전방위(옴니채널) 콘텐츠**가 필요하다.

## B. 노출 전략 — AI가 '발견·이해'하게 만들기

- **B1.** **브랜드명·서브타이틀을 일원화** → AI가 흩어진 콘텐츠를 *같은 매장(엔티티)* 으로 인식.
- **B2.** 홈피에 **AI가 이해하는 FAQ + schema.org 구조화 마크업** 구축 → 질문-답변·구조화 데이터가 AEO의 핵심.
- **B3.** 위 일관된 정보를 **전 채널에 배포**한다.
- **B4. (2026-07 갱신) 노출은 "구글 단독"이 아니라 멀티엔진 GEO다.** ChatGPT·Perplexity·Claude·Google AI Overviews 등 여러 AI 검색에 동시에 걸려야 한다. 노출 레버는 특정 벤더 엔진이 아니라 **schema.org 마크업 + FAQ + 옴니채널 콘텐츠**(→ §D2 루프)다. (시장조사 근거: `docs/GOOGLE_ORDERING_RESEARCH.md`)

## C. 핵심 통찰 — 손님이 곧 콘텐츠 엔진

- **C1.** 매장에 **찾아오는 손님이 최고의 홍보 대사**다.
- **C2.** 그 손님이 **재방문**할 때 비로소 매장은 성공한다.
- **C3.** 전략: **"손님을 (선하게) 활용하자."**

## D. 콘텐츠 생산 메커니즘 — 손님이 만들게 한다

- **D1.** 손님이 **AI 챗봇(샤비)** 으로 리뷰를 쓴다.
- **D2.** **질문-대답(Q&A) 방식**으로 리뷰를 생성 → **AI 검색에 잘 걸리도록**.
- **D3.** 그 리뷰를 **페북·인스타·스레드·링크드인·유튜브 쇼츠·틱톡·구글 비즈니스** 등 **전부 자동 배포**.
- **D4.** 더 나아가, 손님이 **자기 SNS 계정에 매장 리뷰를 올리고 매장 홈피와 연동**해 주면 최상.

## D2. 음성주문 → 콘텐츠 플라이휠 (핵심 심화)

> AEO·GEO 최적화의 **연료는 콘텐츠**이고, 콘텐츠의 **원천은 손님의 주문 그 자체**다.
> 즉 **"주문을 받는 행위"가 곧 "마케팅 콘텐츠를 생산하는 공정"** 이 되도록 설계한다.

- **D2-1. 음성주문 과정 데이터가 1차 원료.** 손님과 **음성으로 대화하며 주문**받는 과정 자체가 가장 값진 데이터다. 이 대화에서 **고객의 취향·패턴·니즈를 파악**하고, 재활용 가능한 **에피소드(스토리 소재)** 를 뽑아낸다.
- **D2-2. 샤비 대화로 콘텐츠화.** 주문 데이터에 **샤비와의 대화**를 얹어 콘텐츠로 가공한다. 대화 중 손님이 **사진 2장 + 15초 영상**을 찍도록 자연스럽게 유도한다.
- **D2-3. 미니홈피 기록 + 스키마 마크업.** 생성된 콘텐츠를 **매장 미니홈피에 기록**하되, **AI 검색에 잘 걸리도록 schema.org 구조화 마크업(AEO/GEO)** 을 함께 심는다.
- **D2-4. 10개 SNS 배포.** 그 콘텐츠를 **10개 SNS 채널**(페북·인스타·스레드·링크드인·유튜브 쇼츠·틱톡·구글 비즈니스 등)에 자동으로 뿌린다.
- **D2-5. 보상 → 기부 → 기부왕.** 사진·영상·리뷰 기여에 **상응하는 스탬프**를 준다. 스탬프는 안 쓰거나 뜻이 있으면 **기부**로 흘려보내고, **'이달의 기부왕'** 을 선정해 선순환을 게임화한다.

```
음성주문(대화) → 고객 패턴·니즈·에피소드 추출
   → 샤비 대화 + 사진 2장·15초 영상 → 콘텐츠 생성
   → 미니홈피 기록 + 스키마 마크업(AEO/GEO)
   → 10개 SNS 배포 → AI 검색 노출
   → (보상) 스탬프 → 기부 → 이달의 기부왕 → (다시) 재방문·주문
```

> **중심축 메모(왓슨):** 우리는 "음성주문 엔진"을 파는 게 아니라, **주문 대화를 AEO/GEO 콘텐츠로 바꾸는 마케팅 루프**를 판다. 시장조사 확인 — 소비자 69%가 "기억하는 개인화"를 선호하고, 경쟁사는 전부 '주문 자동화(인건비 대체)'에 몰려 있어 이 조합은 **경쟁이 비어 있는 화이트스페이스**다.

### D2-6. 오더링 엔진 원칙 (2026-07-04 팀 조사 결론)

왓슨·헌터·알프레드 3팀 조사(`docs/GOOGLE_ORDERING_RESEARCH.md`)의 합의:

- **엔진은 엔진룸에, 얼굴은 샤비.** 오더링 엔진으로 **구글 `Gemini Live API`(네이티브 오디오, 한/영 자동전환, 별도 STT 불필요)** 를 쓰되, **대외 홍보에 "구글/Powered by Google"을 내세우지 않는다** — 손님이 기억할 이름은 언제나 **샤비**. (구글은 점주 계약용 신뢰 배지로만 절제 사용.)
- **폐기:** 구글 **Dialogflow CX 이원화**(비용·락인·결 불일치), **"Order with Google" 직접주문**(2024.7 종료).
- **주문 유입:** 구글에서 주문받지 말고 **Google 비즈니스 프로필의 "주문 링크"를 우리 미니홈피로 소유**(Preferred 뱃지 = 클릭 4배).
- **AEO는 주문과 별 트랙:** 노출은 엔진이 아니라 §B4의 멀티엔진 GEO(schema·FAQ·콘텐츠)가 만든다.
- **가드레일:** 실시간 음성/주문은 **반드시 서버 함수 경유**(클라 키 노출 금지) · 한/영 혼용 STT 정확도가 채택의 급소 → 초기 **텍스트 우선, 음성 보조**.

## E. 동기 설계 — 손님은 왜 움직이나

- **E1.** 손님이 **아무 이유 없이 리뷰를 쓰진 않는다** → **보상**이 필요하다.
- **E2.** 그래서 **스탬프 제도**: **7개를 모으면 현금**으로 전환해 준다.

## F. 경제 모델의 역설 — 이 사업의 차별점

- **F1.** 기존 스탬프 회사도 있지만, **고객은 스탬프를 다 못 채운다는 걸 알아서 포기**한다.
- **F2.** 이렇게 **사라지는 스탬프는 전부 돈**이다 (낙전수입/breakage).
- **F3.** 그럼 점주는 스탬프가 사라져 **돈 안 줘도 되니 이득?** → **아니다.**
- **F4.** 점주는 **약간의 비용**으로 손님이 **단 한 번이라도 더 방문**하면 그게 이득이다.
  (재방문 가치 ≫ 스탬프 원가)

## G. 사회적 가치화 — 스탬프 찍는 행위에 '의미'를 부여

- **G1.** 손님이 스탬프 찍는 행동이 **가치 있는 사회적 행동**이 되어야 한다.
- **G2.** 스탬프의 3가지 출구:
  - ① **단 1개라도 비영리 단체에 기부**
  - ② **친구에게 선물**
  - ③ **본인이 적립금으로 사용**
- **G3.** 목적은 결국 **"고객이 스탬프를 찍게 만드는 것"** 그 자체.

## H. 게임화 — 지속적으로 찍게 만든다

- **H1.** 게임처럼 **ShareChamp 리스트**를 만든다.
- **H2.** **챔피언(고객)** 을 뽑고,
- **H3.** **챔피언 매장**도 리스트업 후 선정한다.

## I. 실행 엔진 — 진짜 핵심은 '매장 서버(직원)'

- **I1.** 스탬프를 찍게 만드는 **핵심은 매장 내 서버(직원)** 다.
- **I2.** 그래서 **스탬프 캐시 발행액의 약 10%를 서버들에게** 준다 → 직원이 적극적으로 권유.

## J. 귀결 — 선한 영향력의 무한 루프

```
손님 방문 → 직원 권유 → 샤비 Q&A 리뷰 → 전채널 자동 배포
   → AI 검색 노출 → 신규/재방문 → (다시) 손님 방문
```

- **J1.** 스탬프를 찍고 → **AEO·GEO 마케팅**이 되고 → **사회 기부**도 되는 **선한 영향력의 무한 루프**.
- **J2.** 수혜 구조:
  - **소상공인**을 위하고,
  - **사회단체**에 도움 (점주가 **5개 중 1개 지정** — 예: 자기 교회, 우리 동네 유기견센터),
  - **고객**은 보상.
- **J3.** 인정: **'좋은 가게' 현판**을 걸어준다.
- **J4.** 점주 추가 혜택: **기부를 많이 하면 영수증을 발급**받아 **세금 공제** 혜택.
